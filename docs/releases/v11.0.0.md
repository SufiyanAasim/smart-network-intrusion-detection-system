# ⚡ NIDS v11.0.0

> Codename **Cipher** — Autonomy

Cipher advances the Smart Network Intrusion Detection System from operator-assisted
triage to **policy-governed autonomous defense**. Models remain evidence
producers; a separate control plane correlates that evidence, evaluates explicit
guardrails, records every decision, and creates reversible response actions.

## Operating modes

| Mode | Behavior |
| --- | --- |
| **Shadow** | Correlates and simulates actions without changing host state; safe default |
| **Approval** | Queues eligible actions for an Administrator decision |
| **Autonomous** | Executes eligible actions only when server policy explicitly enables execution |

Switching the UI to Autonomous does not bypass the server boundary.
`NIDS_AUTONOMY_EXECUTE=true` is also required before a host command can run.

## Smart correlation

* Groups high-confidence evidence by source IP and bounded time window.
* Requires a configurable number of events at or above the risk threshold.
* Preserves source, target, protocol, row count, model risk, and mode as incident
  evidence.
* Produces stable incident identifiers so Streamlit reruns cannot create an
  unbounded duplicate-action stream.

## Adaptive behavior monitoring

Cipher compares recent persisted traffic with older history across risk score,
packet volume, connection count, and SYN-error rate. It reports Stable, Watch,
or Drift status. A strong drift signal recommends an **offline reviewed
retraining run**; production models are never silently replaced.

## Reversible containment

Eligible response actions use validated argv commands and never invoke a shell.
Windows uses a uniquely named `netsh advfirewall` rule; Linux capture hosts use
an `iptables` rule with a unique comment. Every block has:

* configurable TTL and automatic expiry rollback;
* explicit Administrator rollback;
* private-source protection by default;
* a maximum concurrent-block limit;
* execution failure capture without crashing detection;
* an append-only decision and response audit trail.

The original firewall suggestion module remains suggestion-only. Automated
execution is isolated in `src/nids/autonomy.py` and defaults off in local,
Docker, Compose, and Render configuration.

## Autonomous Defense workspace

The new first-class **Autonomy** tab provides:

* operating-mode control for Administrators;
* correlated incident, pending approval, active block, and simulation metrics;
* behavior-drift status and retraining recommendation;
* Administrator approve/deny and rollback controls;
* Viewer-readable incident and response queues;
* full response audit evidence.

## API additions

The read-only API adds:

* `GET /api/autonomy/summary`
* `GET /api/autonomy/incidents?limit=100`
* `GET /api/autonomy/actions?status=pending&limit=100`

These endpoints expose evidence and state only. Mutating decisions remain behind
the authenticated Administrator UI and server policy.

## Configuration

| Variable | Default | Purpose |
| --- | --- | --- |
| `NIDS_AUTONOMY_MODE` | `shadow` | Initial Shadow, Approval, or Autonomous mode |
| `NIDS_AUTONOMY_EXECUTE` | `false` | Server-side host-action gate |
| `NIDS_AUTONOMY_MIN_RISK` | `90` | Minimum row risk for correlated evidence |
| `NIDS_AUTONOMY_MIN_EVENTS` | `3` | Minimum high-risk rows from one source |
| `NIDS_AUTONOMY_WINDOW_SECONDS` | `300` | Correlation window |
| `NIDS_AUTONOMY_BLOCK_TTL_MINUTES` | `30` | Temporary containment duration |
| `NIDS_AUTONOMY_MAX_ACTIVE_BLOCKS` | `5` | Circuit-breaker limit |
| `NIDS_AUTONOMY_ALLOW_PRIVATE` | `false` | Permit private/LAN source execution |

## Release identity

* Package, API, image metadata, Compose tag, feature contract, and active docs
  report `11.0.0`.
* The current codename is **Cipher**.
* v10.0.0 is recorded as **Argus**.
* The version-neutral `nids-history` volume preserves detections, incidents,
  actions, and audit records across upgrades.

## Safety boundary

Cipher is autonomous only inside policies explicitly chosen by the operator. It
does not claim unrestricted self-learning, silent model replacement, or
unbounded response authority. Shadow validation, approval review, least
privilege, and reversible containment remain the recommended deployment path.

## Release summary

**NIDS v11.0.0 ("Cipher")** is the autonomy milestone: detect, correlate,
decide, contain, audit, and roll back under explicit policy.
