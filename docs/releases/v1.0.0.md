# ⚡ NIDS v1.0.0

> Codename *Sentinel* - Foundation

This pre-release establishes the professional foundation for the Network Intrusion Detection System: a maintainable package layout, reusable feature engineering, repeatable model training, automated checks, and deployment-ready repository scaffolding.

Sentinel focuses on making the project understandable, testable, and safe to extend before the stable operational baseline is tagged.

---

## 🏗️ Repository Foundation

The original project layout was reorganized into clear application, model, data, documentation, configuration, script, and test boundaries.

### Improvements

* Moved application code into the `src/nids/` package.
* Separated datasets, trained models, notebooks, configuration, scripts, and tests.
* Replaced fragile working-directory assumptions with relative resource loading.
* Added a consistent structure for future release documentation.
* Established repository files for contribution, security, support, and release governance.

---

## 🧩 Reusable Feature Engineering

Packet-to-feature conversion was separated from the Streamlit interface so detection logic could be exercised independently.

### Improvements

* Extracted feature-engineering logic into `src/nids/features.py`.
* Made the transformation logic importable by the dashboard, scripts, and tests.
* Reduced interface coupling and duplicated implementation details.
* Added unit-test coverage for the core feature calculations.

---

## 🧠 Repeatable Model Training

Model creation was moved into a dedicated command-line workflow.

### Improvements

* Added `scripts/train_models.py` as the model-retraining entry point.
* Standardized where generated model artifacts are stored.
* Kept training responsibilities separate from dashboard startup.
* Documented the relationship between NSL-KDD data and runtime model files.

---

## 🧪 Quality and Automation

Sentinel introduces the first repeatable verification path for the repository.

### Improvements

* Added a pytest suite for feature-engineering behavior.
* Added GitHub Actions checks for linting and tests.
* Created a foundation for pull-request validation.
* Added development dependencies required by the quality workflow.

---

## 📦 Deployment Scaffolding

The repository can now be reproduced outside a single developer workstation.

### Improvements

* Added Docker support for a consistent runtime environment.
* Documented local installation and execution.
* Added dependency manifests for application and development tooling.
* Prepared the codebase for later desktop and cloud packaging work.

---

## 🏗️ Architecture Progress

Sentinel establishes the initial processing path:

**Packet or PCAP → Feature extraction → Model prediction → Streamlit result**

The important architectural change is separation of concerns: interface code presents results, feature code transforms traffic, training scripts create artifacts, and tests verify reusable logic.

---

## ⚠️ Known Limitations

The following limitations remain in NIDS v1.0.0:

* This is a structure-first pre-release rather than the stable operational baseline.
* Detection history is not yet persisted to a database.
* Only the Random Forest and Decision Tree comparison path is available.
* Live feature calculations do not yet use the later rolling connection window.
* Automated alert delivery and operator-response helpers are not included.

---

## 💻 Platform Compatibility

NIDS v1.0.0 is designed for Python 3.11+ environments on Windows, Linux, and macOS.

Live capture remains dependent on operating-system packet permissions and an available capture backend. PCAP-file analysis does not require live-capture privileges.

---

## 🧰 Technology Stack

Sentinel uses the following foundation stack:

* Python 3.11+
* Streamlit
* scikit-learn
* pandas and NumPy
* Scapy
* pytest
* GitHub Actions
* Docker

---

## 🔄 Upgrade Information

* **Breaking changes:** Repository paths and import locations were reorganized.
* **Database migrations:** None
* **Configuration migrations:** Existing local commands may need to use the new `src/` and `scripts/` paths.

Fresh installation from the restructured repository is recommended for this pre-release.
