# ⚡ NIDS v8.0.0

> Codename *Phalanx* - Infrastructure

This grand release adds automated model retraining, role-aware access, a read-only detection API, encrypted history backups, and enterprise alert destinations to the Network Intrusion Detection System.

Phalanx focuses on making the project easier to operate across teams and automation systems while preserving backward compatibility for existing local installations.

---

## 🔄 Model-Retraining Automation

Model artifacts can now be rebuilt through a repeatable GitHub Actions workflow.

### Improvements

* Added `.github/workflows/retrain.yml`.
* Triggers when NSL-KDD data, the training script, or feature engineering changes on `main`.
* Supports manual workflow dispatch.
* Verifies that generated model artifacts can be loaded.
* Uploads retrained models as workflow artifacts.
* Added a `make api` convenience target alongside the existing development commands.

---

## 👥 Multi-User Roles

The dashboard can now distinguish administrative and read-only users.

### Improvements

* Added JSON-based user configuration through `NIDS_AUTH_USERS`.
* Supports `admin` and `viewer` roles.
* Restricts full-history and encrypted-backup actions to administrators.
* Preserves the single-user authentication path as an administrator account.
* Keeps an unconfigured local application backward compatible.
* Centralizes role checks in `src/nids/auth.py`.

---

## 🔌 Read-Only REST API

Detection history is now accessible to scripts and external monitoring tools without scraping the dashboard.

### Improvements

* Added `/health` for service readiness.
* Added `/api/summary` for aggregate history statistics.
* Added `/api/detections` for recent detection records.
* Added `/api/ip/<ip>` for source-IP evidence.
* Added optional bearer-token protection through `NIDS_API_TOKEN`.
* Uses a dependency-free Python HTTP server and the same SQLite source as the dashboard.

---

## 💾 Encrypted History Backup

Administrators can download an encrypted copy of the local detection database.

### Improvements

* Added Fernet encryption through `src/nids/crypto.py`.
* Reads the backup key from `NIDS_DB_ENCRYPTION_KEY`.
* Exposes the encrypted download only when the key is configured.
* Limits the operation to authorized administrators in multi-user mode.
* Leaves the active SQLite database unchanged.

---

## 🔔 Expanded Alert Routing

Critical detections can now reach two additional incident-response destinations.

### Improvements

* Added PagerDuty Events API v2 support.
* Added Microsoft Teams incoming-webhook support.
* Uses `PAGERDUTY_ROUTING_KEY` and `TEAMS_WEBHOOK_URL` configuration.
* Integrates both destinations into the existing cooldown-aware alert dispatcher.
* Preserves Slack, generic webhook, SMTP, sound, and browser notification paths.

---

## 🏗️ Architecture Progress

Phalanx turns the local monitoring application into a role-aware and automation-ready platform:

**Retrain models → Authenticate user or API client → Query shared history → Export encrypted backup → Route critical alerts**

The dashboard and API share the SQLite evidence layer, while authentication, encryption, alerting, and retraining remain independently testable components.

---

## ⚠️ Known Limitations

The following limitations remain in NIDS v8.0.0:

* The REST API is read-only and does not provide an OpenAPI schema.
* Fernet protects downloaded backups, not transparent queries against the active SQLite file.
* API and dashboard processes require coordinated access to the same database path.
* Production deployment still needs mandatory authentication and rate limiting.
* Role configuration is environment-based rather than managed through an administrative interface.

---

## 💻 Platform Compatibility

NIDS v8.0.0 supports Python 3.11+ source environments, Docker execution, and GitHub-hosted retraining jobs.

The API can run wherever the configured SQLite path is accessible. PagerDuty and Teams integrations require outbound HTTPS access and valid service credentials.

---

## 🧰 Technology Stack

Phalanx adds:

* GitHub Actions model-retraining automation
* Python standard-library HTTP server
* SQLite-backed JSON endpoints
* Fernet encryption from `cryptography>=42.0`
* PBKDF2 authentication with admin/viewer authorization
* PagerDuty Events API v2
* Microsoft Teams webhooks

---

## 🔄 Upgrade Information

* **Breaking changes:** None
* **Database migrations:** None
* **Configuration migrations:** Optional `NIDS_AUTH_USERS`, `NIDS_API_TOKEN`, `NIDS_DB_ENCRYPTION_KEY`, `PAGERDUTY_ROUTING_KEY`, and `TEAMS_WEBHOOK_URL`
* **New dependency:** `cryptography>=42.0`

Single-user authentication remains supported, and every new integration stays disabled until its environment configuration is supplied.
