# 🚀 NIDS v8.0.0

Codename: Cipher (grand release — 5 features)

## Release date
2026-07-17

## Highlights
The final release of the v3–v8 roadmap: automation, access control,
programmatic access, encrypted backups, and more alerting reach.

## Added
- **Model-retraining CI pipeline** (`.github/workflows/retrain.yml`) —
  retrains on training-data/feature changes, verifies the models load, and
  uploads them as artifacts. New `make api` target.
- **Multi-user roles/permissions** (`src/nids/auth.py`) — `NIDS_AUTH_USERS`
  JSON defines admin/viewer accounts; viewers are blocked from admin-only
  actions (full-history and encrypted export). Single-user setups remain
  backward compatible.
- **REST API for detections** (`src/nids/api.py`) — dependency-free,
  read-only JSON API (`/health`, `/api/summary`, `/api/detections`,
  `/api/ip/<ip>`), `python -m nids.api`, optional bearer-token auth. See
  [docs/api/api.md](../api/api.md).
- **Encrypted history-db backup** (`src/nids/crypto.py`) — download a
  Fernet-encrypted backup of `data/history.db` when
  `NIDS_DB_ENCRYPTION_KEY` is set.
- **Extra alert integrations** (`src/nids/alerts.py`) — PagerDuty (Events
  API v2) and Microsoft Teams, alongside the existing Slack/webhook/email.

## Changed
- History-tab export now has three columns (CSV / Excel / encrypted backup)
  and is gated to admin users.
- `SECURITY.md` documents roles, the API, and the backup encryption scope.

## Known issues
- The encrypted backup protects the *exported file*; the live SQLite DB is
  still plaintext at rest (transparent DB encryption is on the backlog).
- Role gating is UI-level access control, not a hard security boundary —
  pair it with network controls / a reverse proxy for exposed deployments.

## Compatibility
- Requires Python 3.11+.
- New dependency: `cryptography>=42.0`. Run `pip install -r requirements.txt`.
- SQLite schema unchanged; existing `history.db` works as-is.

## Upgrade instructions
`pip install -r requirements.txt`, then restart. All new features are opt-in
via env vars (`NIDS_AUTH_USERS`, `NIDS_API_TOKEN`, `NIDS_DB_ENCRYPTION_KEY`,
`PAGERDUTY_ROUTING_KEY`, `TEAMS_WEBHOOK_URL`).

## What's next
The Guardian/Security roadmap (Watchtower → Cipher) is complete. Future
work is open-ended — see the backlog in [ROADMAP.md](../../ROADMAP.md).
