# ⚡ NIDS v5.0.0

> Codename *Bulwark* - Data

This release expands Smart Network Intrusion Detection System investigations with complete history exports and source-IP-centered analysis across stored sessions.

Bulwark focuses on making persisted evidence portable, searchable, and useful outside the dashboard's recent-row view.

---

## 📊 Complete History Export

Operators can export the entire SQLite detection history rather than only the most recent rows shown in the interface.

### Improvements

* Added full-history CSV download from the History tab.
* Added Excel `.xlsx` export when `openpyxl` is installed.
* Added `storage.query_all()` as the shared export data source.
* Keeps the Excel control hidden gracefully when the optional dependency is unavailable.
* Preserves all stored sessions in the exported dataset.

---

## 🔎 Source-IP Drill-Down

Historical evidence can now be investigated around a selected source address.

### Improvements

* Added a source-IP selector to the History tab.
* Displays every stored detection associated with the selected IP.
* Summarizes total detections and per-model attack counts.
* Shows the first-seen and last-seen timestamps.
* Added reusable distinct-IP, per-IP query, and per-IP summary storage functions.

---

## 🧹 Interface Compatibility

The release also removes deprecated Streamlit width calls from the dashboard.

### Improvements

* Replaced `use_container_width=True` with `width='stretch'`.
* Removes related deprecation warnings from application logs.
* Keeps tables and charts responsive across the available layout width.

---

## 🏗️ Architecture Progress

Bulwark extends SQLite from persistence into an investigation and interchange layer:

**Store detections → Query full history or one IP → Summarize evidence → Export CSV or Excel**

Dedicated storage functions keep SQL concerns out of the Streamlit interface and make the same evidence reusable by later reports and APIs.

---

## ⚠️ Known Limitations

The following limitations remain in NIDS v5.0.0:

* Excel export requires the optional `openpyxl` package.
* Export files are not yet encrypted.
* Per-IP analysis does not yet include geographic mapping.
* The dashboard does not yet produce a formatted PDF report.
* Investigation remains local to the Streamlit interface; a REST API arrives later.

---

## 💻 Platform Compatibility

NIDS v5.0.0 supports the same Python 3.11+ and Docker environments as Citadel.

CSV export is available with the core installation. Excel export is available wherever `openpyxl>=3.1` is installed and the browser permits file downloads.
