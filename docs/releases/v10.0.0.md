# ⚡ NIDS v10.0.0

> Codename **Argus** — Verification

## Overview

Argus completes the local operator workflow around two questions that must be
answered before a packet is trusted: **which adapter produced it, and what
traffic could that adapter actually see?** The release adds explicit capture
adapter selection, honest device-versus-LAN visibility guidance, a clearer
role-aware authentication experience, and final responsive UI refinement.

The model pipeline, persisted evidence, triage scoring, alerts, exports, and
read-only API remain compatible with Vigil v9.0.0 data.

## Highlights

### Capture interface selection

The Live Capture tab now discovers adapters through Scapy and presents a
friendly selector containing the adapter description and available address.

* Uses Npcap device identifiers on Windows and native interface names on
  Linux/macOS.
* Supports Ethernet, Wi-Fi, VPN, loopback, and virtual adapters reported by the
  active backend.
* Selects Scapy's route-derived default unless `NIDS_CAPTURE_INTERFACE` names a
  different adapter.
* Locks the selector while capture is running, preventing a silent mid-session
  source change.
* Passes the selected identifier directly to `scapy.sniff(iface=...)`.
* Handles an empty adapter list as a visible readiness problem instead of
  starting an ambiguous capture.

### Honest capture scope

Argus explicitly labels native capture as **this device** scope. On a normal
switched Ethernet or Wi-Fi network, the host usually sees its own traffic plus
visible broadcast/multicast frames—not every other device's unicast traffic.

For whole-LAN monitoring, the UI and documentation direct operators to one of
the correct sensor placements:

* managed-switch SPAN/port mirroring;
* a physical network TAP; or
* a sensor running on the gateway, router, or Linux bridge.

Wireshark is optional. Npcap is sufficient for native Windows capture;
Wireshark is useful for deeper manual inspection or producing capture files for
the upload workflow.

### Separate authentication screens

The authentication gate now has dedicated **Sign in** and **Create account**
views rather than combining every action into one form.

* Sign in presents clickable Administrator and Viewer selectors; each opens the
  matching credential form and preselects the configured local username.
* Account role is derived from the authenticated account; a user cannot choose
  or escalate their role on the form.
* Create account is visible only when `NIDS_SIGNUP_ENABLED=true`.
* New self-service accounts always receive the Viewer role.
* Administrator accounts remain configuration-managed through
  `NIDS_AUTH_USERS` or the single-user environment variables.

### Persistent Viewer registration

Optional self-registration uses a dedicated SQLite account store at
`NIDS_AUTH_DB_PATH` (default `data/auth.db`).

* Passwords require at least 12 characters with uppercase, lowercase, number,
  and symbol classes.
* Passwords are stored only as salted PBKDF2-SHA256 hashes.
* Usernames are constrained to a safe 3–32-character set.
* Duplicate checks are case-insensitive across both configured and registered
  accounts.
* Authentication lockout protection also applies to registered accounts.
* Self-registration is disabled by default and explicitly disabled by the
  Render blueprint.

## Interface refinements

* Balanced the canonical circuit-shield mark in the Credits hero, compact
  sidebar, mobile layout, and authentication shell.
* Replaced the vertical-ellipsis header treatment with an enclosed hamburger
  symbol that better communicates a menu.
* Removed Streamlit's intrusive “Press Enter to submit form” helper while
  retaining explicit full-width submit buttons.
* Added a dedicated **Prepare report exports** action and print-friendly page
  rules; Stop Capture never prepares or downloads CSV, PDF, or Print output.
* Added a configurable 2.5-second minimum live refresh so results remain
  readable during busy traffic while packet intake continues normally.
* Rebuilt the sidebar as a compact one-screen control rail with concise access,
  accuracy, and threat-threshold sections.
* Moved notification preferences beside Deploy, placed Credits beside History,
  and made About a secondary action inside the Credits view.
* Reserved independent toolbar lanes for Streamlit's running activity
  indicator, file-change controls, Deploy, Notifications, and the hamburger
  menu.
* Restored the running-person activity indicator without Streamlit's duplicate
  header Stop action, retained Auto rerun in the hamburger menu, and removed
  only the duplicated manual Rerun item.
* Moved the active access badge out of the Credits hero and into the Live
  Capture title row, aligned directly above Stop Capture.
* Matched the role badge to the Stop Capture column width, enlarged the Credits
  identity mark, and italicized the supporting product description.
* Removed temporary instant role access: Administrator and Viewer selectors now
  open their matching username/password form, and About/Access dialogs share a

* Made all six Detection History summary cards identical in width and height.
* Replaced decorative emoji controls with Material icons and changed persisted

## Configuration

New optional settings:

| Variable | Default | Purpose |
|---|---|---|
| `NIDS_CAPTURE_INTERFACE` | Scapy default | Adapter identifier or exact UI label selected initially |
| `NIDS_SIGNUP_ENABLED` | `false` | Enables trusted-local Viewer self-registration |
| `NIDS_AUTH_DB_PATH` | `data/auth.db` | Persistent registered-account database |

Existing `.env`, shell, Docker, and Render precedence is unchanged. The local
source run loads `.env` without overriding values already supplied by the host.

## Deployment changes

* Docker image metadata and tags now use `10.0.0`.
* Compose uses the version-neutral `nids-history` volume and places the optional
  auth store at `/data/auth.db` beside detection history. Upgrades can set
  `NIDS_HISTORY_VOLUME` to an older volume name during migration.
* The Linux capture profile retains only `NET_ADMIN` and `NET_RAW`; Windows
  Npcap remains a native-host capability and is not added to Linux containers.
* Render persists `/data/auth.db` on the existing disk but keeps
  `NIDS_SIGNUP_ENABLED=false`.
* The API health response reports `10.0.0`.

## Compatibility and migration

No manual detection-history migration is required. When the database opens,
legacy decorated verdict values are normalized in place to `Normal` and
`Attack`; models, API consumers, alert configuration, and uploaded captures
remain compatible. Operators should:

1. Rebuild the image or reinstall requirements.
2. Copy the new environment variables from `.env.example` as needed.
3. Select the intended adapter on the first native live-capture run.
4. Leave self-registration disabled unless the installation is trusted and
   local.

## Security model

Argus maintains the existing boundary between Viewer and Administrator:

| Capability | Administrator | Viewer |
|---|---:|---:|
| Dashboard, live capture, and PCAP analysis | Allowed | Allowed |
| Recent history and consensus triage | Allowed | Allowed |
| Complete-history export | Allowed | Restricted |
| Encrypted database backup | Allowed | Restricted |
| Create another Administrator through the UI | Never | Never |

Public deployments should provision accounts from secrets and keep sign-up
disabled. The local database contains password hashes but should still be
protected with normal filesystem permissions and backups.

## Known limits

* NSL-KDD is a research benchmark and does not represent every modern protocol
  or attack family.
* An adapter selector cannot bypass switch isolation or Wi-Fi encryption; whole-
  LAN visibility still requires correct sensor placement.
* Self-registration does not provide email verification or password recovery
  and is therefore intended for trusted local installations.
* Firewall responses remain suggestions and are never applied automatically.
