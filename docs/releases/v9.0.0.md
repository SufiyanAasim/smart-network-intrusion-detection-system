# ⚡ NIDS v9.0.0

> Codename **Vigil** — Operations

This grand release makes the Network Intrusion Detection System cloud-ready and operationally hardened while adding deterministic consensus triage across every available detection model.

Vigil focuses on secure deployment, trustworthy evidence, responsive operator workflows, adaptive presentation, and a clear path from model disagreement to prioritized investigation.

---

## 🧭 Consensus Threat Triage

Every available model contributes to one explainable batch risk result without replacing its original verdict.

### Improvements

* Gives Random Forest, Decision Tree, and optional Isolation Forest one vote per analyzed row.
* Converts the attack-vote ratio into a deterministic 0–100 risk score.
* Assigns Clear, Guarded, Elevated, or Critical triage levels.
* Preserves every raw model verdict for later investigation.
* Persists risk score, level, and vote evidence through a backward-compatible SQLite migration.
* Added a risk-filtered triage queue to the dashboard.
* Added `GET /api/triage` for scripts and SIEM consumers.

---

## ☁️ Secure Cloud Deployment

Vigil adds explicit deployment definitions for container and Render environments.

### Improvements

* Added a Render blueprint with `$PORT` support and persistent detection-history storage.
* Deploys from the `main` branch after repository checks pass.
* Enforces authentication automatically in Render and production-style environments.
* Disables raw live capture in cloud environments while retaining PCAP-file analysis.
* Fails closed when multi-user authentication configuration is invalid.
* Added a five-failed-attempt, five-minute in-process account lockout.

---

## 🐳 Hardened Container Runtime

The container stack now separates safe default operation from explicitly elevated capture workflows.

### Improvements

* Runs the application as a non-root user.
* Supports a read-only root filesystem and writable persistent history volume.
* Added a dynamic health check and graceful shutdown behavior.
* Added an opt-in read-only REST API profile.
* Added an explicit Linux host-capture profile with only the required network capabilities.
* Keeps packet capabilities away from the default dashboard service.
* Added image build and health smoke checks to CI.

---

## 🔐 Authentication Hardening

Production access controls now fail safely under malformed or hostile configuration.

### Improvements

* Rejects invalid multi-user JSON, roles, duplicate usernames, and malformed password hashes.
* Rejects hostile PBKDF2 iteration counts before expensive hashing begins.
* Adds cooldown-backed account lockout after repeated failed sign-ins.
* Preserves administrator and viewer authorization boundaries.
* Uses a brand-customized login interface matching the dashboard identity.

---

## 📁 Safer PCAP Processing

Uploaded traffic analysis now uses file content as its identity and applies an explicit size boundary.

### Improvements

* Content-hashes PCAP uploads to prevent stale same-name cache reuse.
* Rejects files larger than 50 MB by default.
* Provides clear errors for invalid or unreadable captures.
* Retains PCAP analysis when cloud environments disable live sniffing.
* Points users toward bundled sample captures from the upload interface.

---

## 💾 Storage and API Reliability

Vigil strengthens database compatibility, query validation, and concurrent access behavior.

### Improvements

* Migrates existing v8 SQLite databases in place to the v9 triage schema.
* Creates relative database directories safely.
* Normalizes missing numeric values before persistence.
* Uses bounded waits for concurrent SQLite locks.
* Returns API 400 responses for invalid or out-of-range limits.
* Caps API result sizes and validates percent-decoded IP parameters.
* Supports `NIDS_DB_PATH` for packaged and cross-drive history locations.

## 🏗️ Architecture Progress

Vigil completes the planned Guardian/Security release sequence with a deployable, risk-prioritized evidence flow:

**Authenticate → Capture or upload → Extract features → Preserve model votes → Score consensus risk → Persist evidence → Query dashboard/API → Respond**

The default container remains least-privileged, elevated capture is explicit, production authentication is mandatory, and triage augments rather than erases the original model results.

Vigil continues the architectural progression established by previous releases:

* **Sentinel** established the maintainable repository foundation.
* **Vanguard** stabilized live and PCAP analysis with two-model comparison.
* **Watchtower** added rolling features, persistence, alerts, and anomaly detection.
* **Citadel** introduced configurable critical thresholds and temporal trends.
* **Bulwark** made stored evidence exportable and source-IP searchable.
* **Aegis** added geography, reporting, throughput, notifications, and capture readiness.
* **Bastion** added access protection and reviewable firewall suggestions.
* **Phalanx** connected roles, APIs, encrypted backups, retraining, and alert services.
* **Vigil** hardens deployment and prioritizes evidence through consensus triage.

## ⚠️ Known Limitations

The following limitations remain in NIDS v9.0.0:

* Packet-derived NSL-KDD connection features are approximate.
* Models are trained on an older benchmark and require environment-specific validation.
* Risk scores provide research and operator-support evidence, not a production IDS guarantee.
* Cloud deployments support PCAP upload but cannot sniff the host network directly.
* Account lockout state is process-local and resets when the service restarts.
* Fernet protects exported backups rather than transparently encrypting live SQLite queries.
