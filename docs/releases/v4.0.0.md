# ⚡ NIDS v4.0.0

> Codename *Citadel* - Thresholds

This release gives Network Intrusion Detection System operators direct control over critical-alert sensitivity and adds a temporal view of persisted attack activity.

Citadel focuses on making risk thresholds configurable and historical patterns easier to interpret without changing the underlying model artifacts.

---

## ⚙️ Configurable Critical Threshold

The boundary between suspicious and critical batches is no longer hardcoded.

### Improvements

* Added a Critical threshold slider to the dashboard sidebar.
* Preserves a 20% default for backward-compatible behavior.
* Added `CRITICAL_THRESHOLD_PCT` for environment-based defaults.
* Applies the selected value when generating the smart batch summary.
* Continues to classify non-zero results below the critical threshold as suspicious.

---

## 📈 Historical Attack Trends

Persisted model verdicts can now be reviewed over time instead of only as totals and rows.

### Improvements

* Added an “Attacks over time” chart to the History tab.
* Groups Random Forest and Decision Tree attack counts into per-minute buckets.
* Added `storage.query_trend()` as the reusable data source.
* Makes bursts and quiet intervals visible during investigation.
* Keeps trend data tied to the existing SQLite history database.

---

## 🏗️ Architecture Progress

Citadel adds an operator-controlled decision boundary to the existing monitoring flow:

**Persist detections → Aggregate by minute → Compare model trends → Apply selected critical threshold**

Threshold configuration stays in the interface and environment layer, while trend aggregation remains a storage query, preserving separation between presentation and data access.

---

## ⚠️ Known Limitations

The following limitations remain in NIDS v4.0.0:

* The full database cannot yet be exported from the dashboard.
* Historical investigation does not yet include a per-source-IP drill-down.
* The threshold applies to batch attack percentages rather than a consensus risk score.
* Geographic intelligence and PDF reporting are not included.
* Trend granularity is fixed to per-minute buckets.

---

## 💻 Platform Compatibility

NIDS v4.0.0 retains the source and Docker compatibility of Watchtower.

The threshold control works in every Streamlit environment. Historical trends require a writable SQLite history path containing persisted detections.

---

## 🧰 Technology Stack

Citadel continues to use:

* Python 3.11+
* Streamlit
* Altair temporal charts
* SQLite
* Scapy
* scikit-learn detection models
* pandas and NumPy

---

## 🔄 Upgrade Information

* **Breaking changes:** None
* **Database migrations:** None
* **Configuration migrations:** Optional `CRITICAL_THRESHOLD_PCT`; the default remains 20% when unset.

Existing v3.0.0 history is immediately usable by the new trend query.
