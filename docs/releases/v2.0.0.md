# ⚡ NIDS v2.0.0

> Codename *Vanguard* - Baseline

This release marks the first stable operational baseline of the Network Intrusion Detection System, combining live packet capture, PCAP-file analysis, two-model comparison, and model feature-importance views in one Streamlit dashboard.

Vanguard focuses on delivering a usable starting point without changing the repository foundation established in v1.0.0.

---

## 📡 Live Packet Capture

Operators can classify traffic collected directly from an available network interface.

### Improvements

* Added the stable live-capture workflow to the dashboard.
* Converts captured packets into the feature shape expected by the trained models.
* Presents classification results within the same operator workspace.
* Keeps capture and analysis accessible from a dedicated interface tab.

---

## 📁 PCAP File Analysis

Previously captured traffic can be inspected without starting a live sniffing session.

### Improvements

* Added PCAP upload and parsing through Scapy.
* Reuses the same feature and prediction path as live traffic.
* Supports repeatable demonstrations using saved capture files.
* Keeps offline analysis available when live-capture permissions are unavailable.

---

## 🧠 Dual-Model Comparison

Vanguard provides side-by-side evidence from two supervised detection models.

### Improvements

* Displays Random Forest predictions.
* Displays Decision Tree predictions.
* Presents per-model attack counts for the analyzed batch.
* Preserves each model's verdict instead of hiding disagreement.

---

## 🔎 Explainability View

The dashboard exposes model feature importance so users can inspect which inputs influence the trained estimators.

### Improvements

* Added feature-importance visualizations for supported models.
* Connected model artifacts to readable feature names.
* Provides a basic investigation aid beside raw prediction totals.
* Keeps explainability separate from capture and upload workflows.

---

## 🏗️ Architecture Progress

Vanguard stabilizes the first complete operator loop:

**Capture or upload → Transform packets → Compare RF and DT → Review verdicts and importance**

The release deliberately acts as a stable baseline so later milestones can add persistence, alerts, analytics, and stronger response controls without redefining the core workflow.

---

## ⚠️ Known Limitations

The following limitations remain in NIDS v2.0.0:

* Detection history exists only in the current application session.
* Connection-window statistics still use the baseline approximation.
* There is no third anomaly-detection model.
* Critical detections do not yet trigger external alert channels.
* The dashboard does not yet provide configurable thresholds or historical trend charts.

---

## 💻 Platform Compatibility

NIDS v2.0.0 supports source execution on Python 3.11+ environments.

* Windows live capture requires an appropriate packet-capture driver.
* Linux and macOS live capture require suitable raw-socket privileges.
* PCAP-file analysis remains available without a live-capture session.

---

## 🧰 Technology Stack

Vanguard uses:

* Python 3.11+
* Streamlit
* Scapy
* scikit-learn Random Forest and Decision Tree models
* pandas and NumPy
* joblib model artifacts
* pytest and GitHub Actions
* Docker
