# ⚡ NIDS v6.0.0

> Codename *Aegis* - Analytics

This grand release adds geographic context, formatted PDF evidence, rolling traffic throughput, operator notifications, and capture-driver readiness checks to the Smart Network Intrusion Detection System.

Aegis focuses on making detection activity easier to interpret while giving operators clearer feedback about traffic volume, alert urgency, and host capture readiness.

---

## 🌍 Geographic Intelligence

Stored source addresses can now be categorized and, when optional data is available, plotted geographically.

### Improvements

* Added private, public, loopback, and reserved IP-type classification.
* Added an IP-type breakdown to the History tab.
* Added optional MaxMind GeoLite2-City lookup support.
* Plots resolved public sources on a world map.
* Shows actionable setup guidance when the optional database is unavailable.

---

## 📄 PDF Threat Reports

Analyzed batches can now be exported as concise, shareable PDF summaries.

### Improvements

* Added reportlab-based PDF generation.
* Summarizes packet totals and per-model attack counts.
* Includes the highest-frequency attacker IP addresses.
* Places the PDF download beside the existing CSV output.
* Keeps report creation isolated in `src/nids/reporting.py` for direct testing.

---

## 📊 Real-Time Throughput

The Live Capture workspace now shows traffic volume over a rolling interval.

### Improvements

* Added packets-per-second measurement.
* Added kilobytes-per-second measurement.
* Displays both metrics across a rolling 60-second window.
* Keeps throughput aggregation in `src/nids/throughput.py`.
* Provides traffic context beside detection results.

---

## 🔔 Operator Notifications

Critical events can draw attention even when the operator is not focused on the result banner.

### Improvements

* Added an opt-in synthesized alert sound.
* Added opt-in browser desktop notifications.
* Reuses alert cooldown behavior to limit repeated noise.
* Exposes notification controls in the dashboard sidebar.
* Preserves existing webhook and email delivery paths.

---

## 🧰 Capture Readiness

The application now checks whether the host is prepared for live capture before enabling the workflow.

### Improvements

* Added platform-aware capture-provider detection.
* Shows an Npcap installation link when Windows lacks the required driver.
* Disables Start Capture instead of silently returning no traffic.
* Provides operating-system-specific remediation guidance.
* Keeps PCAP-file analysis available when live capture is not ready.

---

## 🏗️ Architecture Progress

Aegis expands the investigation path with independent analytics adapters:

**Capture readiness → Live traffic → Throughput and detection → Geo context → Notify operator → Export PDF**

Geography, reporting, throughput, notifications, and platform checks remain separate modules so each capability can be tested without importing Streamlit.

---

## ⚠️ Known Limitations

The following limitations remain in NIDS v6.0.0:

* Detailed city mapping requires both `geoip2` and a separate MaxMind database.
* Private and reserved addresses cannot be placed on the public world map.
* Browser notifications depend on browser permission and support.
* Npcap installation is external to the Python application.
* PDF reports summarize a classified batch rather than the complete database.

---

## 💻 Platform Compatibility

NIDS v6.0.0 supports Python 3.11+ source environments and Docker execution.

* Windows live capture uses Npcap.
* Linux and macOS use their available raw-socket or libpcap paths with appropriate privileges.
* Browser sound and desktop notifications depend on the client browser.
* Optional MaxMind mapping works wherever a readable GeoLite2-City database is configured.

---

## 🧰 Technology Stack

Aegis adds:

* reportlab for PDF generation
* geoip2 for optional location lookup
* MaxMind GeoLite2-City data supplied by the operator
* Streamlit and Altair for analytics presentation
* Scapy for capture
* SQLite for persisted evidence
* Browser Notification APIs for desktop alerts

---

## 🔄 Upgrade Information

* **Breaking changes:** None
* **Database migrations:** None
* **Configuration migrations:** Optional `GEOIP_DB_PATH` for geographic lookup
* **New dependency:** `reportlab>=4.0`
* **Optional dependency:** `geoip2>=4.7`

Existing detection history is immediately usable for IP-type analytics and PDF-supported workflows.
