# 🚀 NIDS v7.0.0

Codename: Bastion

## Release date
2026-07-17

## Highlights
Hardens the dashboard for shared use: an optional login gate, and
ready-to-apply firewall block suggestions for flagged attackers.

## Added
- **Dashboard auth/login** (`src/nids/auth.py`) — an optional PBKDF2-SHA256
  password gate. Generate a hash with `python -m nids.auth`, set
  `NIDS_AUTH_PASSWORD_HASH` (and optionally `NIDS_AUTH_USERNAME`) in `.env`,
  and the app shows a login form until authenticated, with a sidebar logout.
  Passwords are never stored or compared in plaintext. Unconfigured = open
  (backward compatible).
- **Auto-block suggestion** (`src/nids/firewall.py`) — a "🚫 Suggested block
  rules" expander offers copy-paste firewall commands (iptables, ufw,
  nftables, Windows netsh) for a flagged attacker IP, shown in the
  critical-threat summary and the per-IP drill-down. Suggestion-only: the
  app never runs these or changes system state.

## Changed
- Sidebar shows the signed-in user and a logout button when auth is active.
- `SECURITY.md` documents the login threat model and the
  suggestion-only nature of block rules.

## Fixed
- N/A

## Known issues
- The login is a lightweight gate for small/internal deployments — put the
  app behind a reverse proxy / SSO + HTTPS for anything internet-facing.

## Compatibility
- Requires Python 3.11+. No new third-party dependencies (both features are
  standard-library only). SQLite schema unchanged.

## Upgrade instructions
No migration needed. To enable login, generate a hash with
`python -m nids.auth` and set `NIDS_AUTH_PASSWORD_HASH` in `.env`.
