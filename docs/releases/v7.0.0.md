# ⚡ Smart Network Intrusion Detection System v7.0.0

> Codename *Bastion* - Security

This release adds an optional password gate and operator-reviewed firewall suggestions to the Smart Network Intrusion Detection System.

Bastion focuses on protecting dashboard access and shortening the path from a flagged source address to a practical response command without allowing the application to modify the host firewall automatically.

---

## 🔒 Dashboard Authentication

Administrators can require a username and password before exposing dashboard content.

### Improvements

* Added an optional login form that halts dashboard rendering until authentication succeeds.
* Uses PBKDF2-SHA256 password hashing from the Python standard library.
* Reads the password hash from `NIDS_AUTH_PASSWORD_HASH` rather than storing plaintext credentials.
* Supports an optional `NIDS_AUTH_USERNAME` value.
* Added a sidebar logout control for authenticated sessions.
* Keeps local installations backward compatible by running openly when authentication is not configured.

---

## 🚫 Firewall Rule Suggestions

Flagged source addresses now include ready-to-review host firewall commands.

### Improvements

* Generates iptables rules for Linux environments.
* Generates ufw commands for compatible Linux hosts.
* Generates nftables rules.
* Generates Windows `netsh advfirewall` commands.
* Displays suggestions in critical summaries and per-IP investigation views.
* Keeps every command copyable for explicit operator review and execution.

---

## 🛡️ Safe Response Boundaries

Bastion deliberately separates detection support from system mutation.

### Improvements

* The application never executes generated firewall commands.
* No host firewall state is changed automatically.
* Operators retain responsibility for validating the address, direction, and operational impact.
* Suggestions are generated by the isolated `src/nids/firewall.py` module.
* Authentication and firewall behavior have dedicated automated tests.

---

## 🏗️ Architecture Progress

Bastion places an access boundary before the existing operator workflow and adds a reviewable response output after detection:

**Authenticate → Analyze traffic → Flag source → Generate suggestion → Operator reviews and applies**

Authentication state remains in the application session, while password hashing and rule generation stay in independent standard-library modules.

---

## ⚠️ Known Limitations

The following limitations remain in Smart Network Intrusion Detection System v7.0.0:

* The v7 login gate is optional and single-user.
* There is no failed-attempt lockout in this release.
* Generated firewall commands are suggestions, not automatic containment.
* Operators must confirm that a flagged address is safe to block.
* Public deployment still requires HTTPS, secret management, and network-level protection outside the application.

---

## 💻 Platform Compatibility

Smart Network Intrusion Detection System v7.0.0 supports the same Python 3.11+ source and Docker environments as Aegis.

Firewall suggestions are emitted for common Windows and Linux tooling, but availability and syntax support depend on the host firewall. The dashboard does not request elevated privileges to apply them.

---

## 🧰 Technology Stack

Bastion continues the existing stack and adds no third-party dependency for its two headline features:

* Python `hashlib`, `hmac`, and `secrets`
* PBKDF2-SHA256 password hashing
* Streamlit session state
* Standard-library IP validation
* Platform-specific firewall command templates
* pytest security and response tests

---

## 🔄 Upgrade Information

* **Breaking changes:** None
* **Database migrations:** None
* **Configuration migrations:** Optional `NIDS_AUTH_USERNAME` and `NIDS_AUTH_PASSWORD_HASH`
* **New third-party dependencies:** None

Existing installations remain open until an authentication hash is configured.
