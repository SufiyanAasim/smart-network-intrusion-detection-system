# ⚡ Smart Network Intrusion Detection System v3.0.0

> Codename *Watchtower* - Consolidation

This release consolidates the Smart Network Intrusion Detection System into a feature-rich monitoring workspace with real connection-window statistics, durable SQLite history, multi-channel alerts, a third anomaly model, and a cohesive branded interface.

Watchtower focuses on preserving more evidence, improving detection context, and making sustained monitoring practical beyond a single in-memory session.

---

## 📡 Windowed Traffic Features

Traffic features now use recent packet history instead of treating every packet as an isolated connection.

### Improvements

* Added trailing two-second and 100-connection windows.
* Computes `count`, `srv_count`, error-rate, and same-service-rate fields from real recent traffic.
* Keys window calculations to packet capture timestamps.
* Maintains a rolling raw-packet buffer during live capture.
* Replaces the earlier single-packet approximation for window-dependent fields.

---

## 💾 Persistent Detection History

Classified results now survive application reruns and workstation restarts.

### Improvements

* Added SQLite-backed storage in `src/nids/storage.py`.
* Persists every classified detection beyond the 100-row interface buffer.
* Added a History tab with totals and the latest 200 detections.
* Added source filtering for live and uploaded traffic.
* Records detection evidence in a queryable local database.

---

## 🔔 Critical-Threat Alerting

Operators can route critical detection summaries outside the dashboard.

### Improvements

* Added Slack incoming-webhook alerts.
* Added generic webhook delivery.
* Added SMTP email delivery.
* Added per-model cooldown throttling to reduce repeated notifications.
* Added Streamlit toast feedback alongside the in-page critical banner.

---

## 🧠 Third Detection Model

Watchtower expands model comparison with an unsupervised anomaly signal.

### Improvements

* Added Isolation Forest training and runtime inference.
* Displays the anomaly verdict beside Random Forest and Decision Tree results.
* Degrades gracefully to the two original models when the optional artifact is absent.
* Refactored model-column rendering into a shared interface helper.

---

## 🎨 Interface and Documentation

The dashboard now presents a consistent security-focused identity and clearer session context.

### Improvements

* Added the shield logo to the application and README.
* Added a custom dark Streamlit theme.
* Added an About panel and live version badge.
* Added a session packet counter.
* Documented the NSL-KDD source, citation, file layout, and feature columns.

---

## 🏗️ Architecture Progress

Watchtower expands the processing sequence into a persistent monitoring loop:

**Capture history → Windowed features → Three-model analysis → Persist evidence → Alert operator**

SQLite becomes the durable source for investigation, while alert adapters remain isolated from feature computation and interface rendering.

---

## ⚠️ Known Limitations

The following limitations remain in Smart Network Intrusion Detection System v3.0.0:

* The critical threshold is still fixed rather than operator-configurable.
* History shows a recent window but does not yet provide full CSV or Excel export.
* Per-IP historical drill-down is not yet available.
* GeoIP mapping, PDF reports, and throughput charts are planned for later releases.
* Isolation Forest does not expose the same feature-importance interface as the supervised models.

---

## 💻 Platform Compatibility

Smart Network Intrusion Detection System v3.0.0 supports Python 3.11+ source environments and Docker-based execution.

Live packet capture remains subject to operating-system privileges and capture-backend availability. Slack, generic webhook, and SMTP features require their corresponding environment configuration.

## 🔄 Upgrade Information

* **Breaking changes:** None
* **Database migrations:** A new SQLite history database is created automatically.
* **Configuration migrations:** Alert environment variables are optional; existing installations continue without external alerts.

Retrain models to generate `models/iforest_model.pkl` when the third comparison model is required.
