# 🚀 NIDS v3.0.0

Codename: Watchtower

## Release date
2026-07-16

## Highlights
First feature release on top of the v2.0.0 (Vanguard) baseline: real
windowed feature computation, persistent detection history, critical-threat
alerting, a third (unsupervised) comparison model, and a full UI/UX/branding
pass — plus dataset documentation.

## Added
- Real trailing 2s/100-connection windowed `count`/`srv_count`/`*serror_rate`/
  `same_srv_rate` computation (`src/nids/features.py`), replacing the
  hardcoded single-packet approximation.
- `src/nids/storage.py` — SQLite-backed detection history (`data/history.db`)
  beyond the in-memory 100-row buffer, with a new "📜 History" tab
  (totals, recent detections, source filter).
- `src/nids/alerts.py` — critical-threat alerting via Slack webhook, generic
  webhook, or SMTP email, cooldown-throttled per model.
- `src/nids/anomaly.py` + Isolation Forest as a third, unsupervised
  comparison model alongside Random Forest and Decision Tree.
- Project logo (`assets/images/logo.svg`), custom dark theme, sidebar
  branding, "About this project" panel, toast alerts, live packet counter.
- `docs/DATASET.md` — NSL-KDD source, citation, file layout, column meanings.

## Changed
- Live-capture loop keeps a rolling raw-packet buffer so windowed stats have
  real history to compute from.
- `display_results` refactored to a shared `render_model_column` helper
  (removes duplicated RF/DT rendering, scales to a 3rd model column).
- `scripts/train_models.py` fixed to run standalone (was missing the `src/`
  import bootstrap) and now also trains/saves `models/iforest_model.pkl`.

## Known issues
- Live-capture windowed features are still an approximation of true NSL-KDD
  connection records (see docs/architecture/architecture.md).
- No CI smoke test yet that boots `streamlit run` end-to-end (tracked for v4.0.0).

## Compatibility
- Requires Python 3.11+.
- `models/*.pkl` retrained during this release; if you have older model
  files from v2.0.0, re-run `python scripts/train_models.py`.

## Upgrade instructions
Pull, then `pip install -r requirements.txt` (no new dependencies were
added — alerts/storage use the standard library + existing pandas/sqlite3).
